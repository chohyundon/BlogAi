## PR 작성 지침 (에이전트용)

현재 브랜치의 **커밋·`main`(또는 베이스 브랜치) 대비 diff**를 기준으로 PR 초안을 작성할 때 아래를 따른다.

에이전트에게 요청할 때는 채팅에 **`@pr_agents.md`** 를 붙인다.

---

### PR 제목 형식

```
[<project_name>] <type>(<scope>): <subject>
```

- **`project_name`**: 이 레포는 **`self`** 를 쓴다 (저장소 표기가 필요하면 팀 규칙에 따라 `Likelion` 등으로 통일).
- **`type`**, **`scope`**, **`subject`**: 프로젝트 **Git 커밋 컨벤션**과 동일하게 맞춘다.
- 로컬 커밋 메시지에 붙는 `| GitHub ID | HH:MM` 은 **PR 제목에는 넣지 않는다.**

**제목 예시**

```
[self] docs(agent): 에이전트 PR 지침 pr_agents.md 추가
```

---

### PR 본문 형식

`pull_request_template.md` 와 같은 의도로, 아래 **5개 섹션**을 순서대로 채운다.

| 순서 | 섹션        | 작성 내용                            |
| ---- | ----------- | ------------------------------------ |
| 1    | 변경 내용   | 바뀐 파일·동작·삭제·추가를 요약      |
| 2    | 변경 이유   | 문제 정의 또는 목적                  |
| 3    | 주요 구현   | 핵심 코드 경로, 설정, 스키마 등      |
| 4    | 영향 범위   | 사용자 플로우, API, DB, 배포, 호환성 |
| 5    | 테스트 여부 | 실행한 명령·결과, 또는 미실행 사유   |

본문 **마지막**에 항상 다음 체크리스트를 붙인다.

```
- [ ] pnpm lint
- [ ] pnpm test
```

---

### 예시 PR 본문 (에이전트 출력 참고용)

아래는 **문서-only 변경**일 때 채울 수 있는 예시다. 실제 PR에서는 현재 브랜치 diff에 맞게 바꾼다.

#### 1. 변경 내용

- 프로젝트 루트에 **`pr_agents.md`** 를 추가했다.
- 기존 **`AGENTS.md`** 는 제거하고, 에이전트용 PR 제목 규칙·본문 5섹션·체크리스트를 한 파일로 정리했다.

#### 2. 변경 이유

- PR 설명 형식을 레포 안에서 통일하기 위해.
- `@pr_agents.md` 로 에이전트에게 동일한 템플릿을 주기 위해.

#### 3. 주요 구현

- 애플리케이션 코드 변경 없음. 마크다운 문서만 추가·교체.

#### 4. 영향 범위

- 런타임·API·DB에는 영향 없음. 개발 시 PR/에이전트 작성 워크플로만 해당.

#### 5. 테스트 여부

- 문서 변경만 해당. `pnpm lint`, `pnpm test` 는 PR 체크리스트에서 확인 예정.

```
- [ ] pnpm lint
- [ ] pnpm test
```
